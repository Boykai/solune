## Entities

### ActivityEvent (Existing — No Schema Changes)

The `activity_events` table stores all activity log entries. New event types reuse the existing schema as new string values in the `event_type` and `action` columns (FR-016).

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| `id` | `TEXT PRIMARY KEY` | `activity_events.id` | UUID, auto-generated by `log_event` |
| `event_type` | `TEXT NOT NULL` | `activity_events.event_type` | Extended with `"settings"`, `"project"` values |
| `entity_type` | `TEXT NOT NULL` | `activity_events.entity_type` | Extended with `"settings"`, `"project"` values |
| `entity_id` | `TEXT NOT NULL` | `activity_events.entity_id` | Entity identifier (project ID, pipeline ID, etc.) |
| `project_id` | `TEXT NOT NULL` | `activity_events.project_id` | Scoping — all events belong to a project |
| `actor` | `TEXT NOT NULL` | `activity_events.actor` | Username or `"system"` |
| `action` | `TEXT NOT NULL` | `activity_events.action` | Extended with `"launched"`, `"triggered"`, `"completed"`, `"selected"`, `"pr_merged"`, `"copilot_pr_ready"` |
| `summary` | `TEXT NOT NULL` | `activity_events.summary` | Human-readable one-liner |
| `detail` | `TEXT` (JSON) | `activity_events.detail` | Structured metadata (changed_fields, issue_number, agent_count, etc.) |
| `created_at` | `TEXT NOT NULL` | `activity_events.created_at` | ISO 8601 timestamp, auto-set |

**New Event Type Values**:

| event_type | action | entity_type | Trigger Point | detail Contents |
|------------|--------|-------------|---------------|-----------------|
| `settings` | `updated` | `settings` | Settings PUT endpoints | `{"scope": "user\|global\|project", "changed_fields": [...]}` |
| `project` | `created` | `project` | Project creation endpoint | `{"project_name": "..."}` |
| `project` | `selected` | `project` | Project selection endpoint | `{"project_name": "..."}` |
| `pipeline_run` | `launched` | `pipeline` | `execute_pipeline_launch()` | `{"issue_number": N, "agent_count": N, "pipeline_name": "..."}` |
| `agent_execution` | `completed` | `pipeline` | Orchestrator `handle_completion()` | `{"workflow_id": "...", "pipeline_name": "..."}` |
| `agent_execution` | `triggered` | `agent` | Orchestrator `assign_agent_for_status()` | `{"agent_name": "...", "status": "..."}` |
| `webhook` | `pr_merged` | `issue` | Webhook handler (PR merged) | `{"pr_number": N, "repository": "..."}` |
| `webhook` | `copilot_pr_ready` | `issue` | Webhook handler (Copilot PR) | `{"pr_number": N, "repository": "...", "branch": "..."}` |

### ActivityStats (New — Response Model)

Aggregated summary of activity events returned by `GET /activity/stats`. This is a read-only view computed server-side via SQL (FR-009).

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `total_count` | `int` | Yes | Total number of events for the project |
| `today_count` | `int` | Yes | Number of events in the last 24 hours |
| `by_type` | `dict[str, int]` | Yes | Event counts grouped by `event_type` for the last 7 days |
| `last_event_at` | `str \| None` | Yes | ISO 8601 timestamp of the most recent event, or `null` if no events |

**Pydantic Model** (to be added to `src/models/activity.py`):

```python
class ActivityStats(BaseModel):
    """Response model for activity summary statistics."""

    total_count: int
    today_count: int
    by_type: dict[str, int]
    last_event_at: str | None
```

**Derived Frontend Values** (computed from `ActivityStats` in the UI):

| Stat Card | Source | Computation |
|-----------|--------|-------------|
| "Total Events" | `total_count` | Direct display |
| "Today" | `today_count` | Direct display |
| "Most Common" | `by_type` | Key with highest value in `by_type` dict |
| "Last Activity" | `last_event_at` | Relative time formatting (e.g., "5m ago") |

### Time Bucket (Frontend-Only — No Persistence)

Client-side grouping of events by time period. Pure utility function, no model or state persistence.

| Bucket | Condition | Display |
|--------|-----------|---------|
| Today | `created_at` is today (local timezone) | "Today" sticky header |
| Yesterday | `created_at` is yesterday | "Yesterday" sticky header |
| This Week | `created_at` is within the last 7 days (excluding today/yesterday) | "This Week" sticky header |
| Earlier | `created_at` is older than 7 days | "Earlier" sticky header |

## State Transitions

This feature does not introduce new stateful entities. Activity events are append-only records. The stats endpoint computes aggregations on-the-fly from the existing event data.

The logging flow for each new event point:

```text
[Primary Operation Succeeds]
    │
    └── log_event(db, event_type="...", action="...", ...)
            │
            ├── Success → row inserted, committed
            └── Failure → exception caught, logged as WARNING, operation unaffected
```

The stats query flow:

```text
[GET /activity/stats?project_id=X]
    │
    └── get_activity_stats(db, project_id=X)
            │
            ├── Query 1: SELECT COUNT(*), MAX(created_at) → total_count, last_event_at
            ├── Query 2: SELECT COUNT(*) WHERE created_at >= now-24h → today_count
            └── Query 3: SELECT event_type, COUNT(*) WHERE created_at >= now-7d GROUP BY → by_type
            │
            └── Return ActivityStats(total_count, today_count, by_type, last_event_at)
```
